# 流云 Windows 待办事项应用设计规格

日期：2026-06-26
状态：待用户审阅

## 目标

“流云”第一版是一个仅本地使用的 Windows 待办事项桌面应用。它用于记录带日期、时间、地点、分类和提醒规则的待办事项，并在指定时间弹出自定义提醒窗口。

第一版聚焦 Windows。Android 版本放到后续阶段，数据通过手动导入/导出文件同步，不使用联网服务。云同步和插件系统不是本项目目标，后续阶段也不规划。

## 设计基准

用户提供的 `D:\Codex文件\Web-Prototype\` 静态原型作为第一版视觉基准。正式实现沿用以下方向：

- 左侧导航、顶部栏、中间日期分组列表、右侧日历概览。
- 温和纸质感背景、低对比边框、墨蓝主强调色。
- 设置页包含背景图片/GIF、透明度、字体、主题色、开机启动、导入导出。
- 新建/编辑待办使用弹窗表单。

正式实现不直接复制多份内联 CSS，而会抽取统一主题变量、组件样式和页面组件。

## 技术方案

推荐技术栈：

- 桌面壳：Tauri
- 前端：React + TypeScript
- 本地数据库：SQLite
- 后端逻辑：Tauri Rust commands
- 样式：CSS variables + 组件样式
- 字体：内置开源中文字体资源

选择原因：

- Tauri 安装包和运行占用比 Electron 更轻。
- React 能复用现有 HTML 原型的结构和样式思想。
- SQLite 适合本地离线数据、重复提醒和导入导出。
- Tauri 后端能处理本地文件、数据库、开机启动和多窗口提醒。

不使用本机 MySQL 作为应用主数据库。虽然当前电脑已安装 MySQL，但它更适合服务端或多人系统；本项目是单机离线桌面软件，使用 MySQL 会增加安装配置、后台服务、端口、账号密码、备份迁移和未来 Android 兼容成本。SQLite 随应用内置数据文件运行，更适合本地个人工具。

## 模块划分

### todo

负责待办事项增删改查、完成状态、优先级、分类、标签和地点。

### repeat

负责重复规则和下一次提醒时间计算。第一版支持：

- 不重复
- 每天
- 每周
- 每月

暂不支持每 N 天、工作日、每月最后一天等复杂规则。

### reminder

负责提醒调度和弹窗控制：

- 应用运行时每 30 秒扫描一次到期提醒。
- 到期后打开自定义提醒窗口。
- 支持完成、稍后提醒、打开详情、关闭。
- 稍后提醒第一版固定为 10 分钟。

### settings

负责用户设置：

- 开机启动
- 内置字体选择和字号
- 背景图片/GIF
- 背景透明度
- 主题色
- 自动备份策略

### storage

负责 SQLite 数据库、配置文件、资源文件路径、背景图片复制和备份文件管理。

### import-export

负责 JSON/CSV 导入导出：

- JSON 是完整备份格式，包含待办、分类、标签、设置和重复规则。
- CSV 主要用于表格软件查看，不作为完整恢复格式。
- 导入时默认合并数据，不覆盖已有条目。

### ui

负责主界面、分类页、统计页、设置页、新建/编辑弹窗和自定义提醒弹窗。

## 数据模型

### Todo

核心字段：

- `id`
- `title`
- `description`
- `location`
- `due_at`
- `priority`: `high | medium | low`
- `category_id`
- `status`: `pending | done | overdue`
- `reminder_enabled`
- `reminder_offset_minutes`
- `repeat_rule`
- `next_reminder_at`
- `last_reminded_at`
- `created_at`
- `updated_at`
- `completed_at`

### RepeatRule

第一版可存为结构化 JSON：

- `type`: `none | daily | weekly | monthly`
- `anchor_time`
- `anchor_day`

### Settings

核心字段：

- `launch_on_startup`
- `font_family`
- `font_size`
- `theme_color`
- `background_image_path`
- `background_opacity`
- `auto_backup_enabled`
- `backup_retention_days`

### ReminderEvent

用于记录提醒历史，避免同一提醒点重复打扰：

- `id`
- `todo_id`
- `scheduled_at`
- `shown_at`
- `action`: `shown | completed | snoozed | dismissed | opened`

## 提醒行为

应用运行时：

1. 每 30 秒查询 `next_reminder_at <= now`、`reminder_enabled = true`、`status != done` 的待办。
2. 对尚未在当前提醒点弹出过的任务打开提醒窗口。
3. 非重复任务提醒后记录历史，不在同一提醒点无限重复弹出。
4. 重复任务在用户点击“完成”后计算下一次提醒时间。
5. 用户点击“稍后提醒”时，将下一次提醒时间设置为当前时间后 10 分钟。

开机启动：

- 用户开启设置后，Windows 开机时自动启动应用。
- 根据用户确认，开机后自动打开主程序窗口。
- 主窗口显示今天和近期待办。
- 启动后检查错过提醒，并在主窗口顶部显示错过提醒提示。

## 自定义提醒弹窗

提醒弹窗是独立 Tauri 窗口，不使用普通 Windows 系统通知。

内容：

- 任务标题
- 日期时间
- 地点
- 描述
- 分类和优先级

样式：

- 读取设置中的背景图片/GIF。
- 读取设置中的字体、字号和透明度。
- 弹窗置顶，但不全屏。
- 支持图片和 GIF 背景展示。

按钮：

- 完成
- 稍后提醒
- 打开详情
- 关闭

## 字体方案

正式程序内置开源中文字体，不依赖系统已安装字体。

默认建议：

- `LXGW WenKai / 霞鹜文楷`

原因：

- 中文观感自然。
- 与当前原型的纸质、书写感方向一致。
- 可作为程序资源随安装包分发。

实现方式：

- 字体文件放入前端资源目录。
- 通过 `@font-face` 注册。
- 设置页仍保留“系统默认字体”作为备用选项。

## 界面设计

### 主界面

- 左侧导航：全部待办、今日任务、即将到来、已逾期、分类、统计、设置。
- 顶部栏：页面标题、当前日期、搜索、新建待办。
- 中间列表：按日期分组展示待办。
- 待办条目显示：时间、标题、描述、地点、优先级、分类、重复标记。
- 右侧面板：迷你日历、今日概览、逾期数量、完成数量。

### 新建/编辑待办弹窗

字段：

- 标题
- 描述
- 地点
- 日期
- 时间
- 优先级
- 分类
- 提醒开关
- 提前提醒
- 重复规则

### 设置页

功能：

- 背景图片/GIF 上传和移除
- 背景透明度滑块
- 字体和字号选择
- 主题色选择
- 开机启动开关
- 数据自动备份设置
- JSON 导入/导出
- CSV 导出

### 分类页

第一版支持基础分类展示和选择。分类管理可以在第二阶段完善。

### 统计页

第一版可保留页面结构，统计数据在第二阶段完善为真实计算。

## 分阶段实现

### 第一阶段：Windows MVP

- 创建 Tauri + React + SQLite 项目。
- 实现主界面、设置页、新建/编辑待办弹窗。
- 支持本地新增、编辑、删除、完成待办。
- 支持地点、日期时间、优先级、分类、提醒开关。

- 通过手动导入/导出同步数据。

## 验证计划

界面模块：

- 启动本地应用，检查主界面、设置页、弹窗布局。
- 检查不同窗口尺寸下文本不重叠、不溢出。

数据模块：

- 新增、编辑、删除、完成待办。
- 检查重启应用后数据仍存在。

提醒模块：

- 创建 1 分钟后的测试提醒，确认弹窗出现。
- 测试完成、稍后提醒、打开详情、关闭。
- 测试非重复任务不会在同一提醒点反复弹出。

重复任务：

- 测试每天、每周、每月的下一次提醒计算。
- 测试完成重复任务后自动更新下一次提醒时间。

设置模块：

- 测试字体、字号、主题色保存。
- 测试背景图片/GIF 和透明度保存。
- 测试开机启动开关写入成功。

导入导出：

- 导出 JSON 后重新导入。
- 确认标题、描述、地点、日期时间、提醒规则、重复规则不丢失。

## 非目标

本项目不做：

- 云同步。
- 用户账号。
- 自动联网同步 Android。
- 复杂重复规则。
- 多人协作。
- 插件系统。
- 依赖本机 MySQL 服务作为主数据库。
